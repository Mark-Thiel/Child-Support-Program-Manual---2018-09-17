Ways In Which Data Enters Case Records in ASSETS

 

 

Ways in which data can enter a child support case record:

    1. WMS pre-populates some datafields in ASSETS when a CSEU worker builds a case through Referral Search.

    2. CSEU workers manually input data into the case record.

    3. ASSETS imports data from WMS when a Non-TA case is converted to a TA case.

    4. WMS updates ASSETS, on an ongoing basis, when certain criteria are met â€” for example, when a residential address is updated in WMS by a TA Worker.  (For more information, see How and when does WMS change a CP address in ASSETS?)

    5. FPLS, SPLS, and APRRS automatically populate and/or update datafields in ASSETS when data is found through automated location searches.

    6. Workers at the Processing Center manually enter data into their computer system, which then updates ASSETS during the next batch processing (data exchange) session.

    7. Data is captured at the Processing Center when paper mail arrives there.  All paper mail is scanned and imaged upon arrival. Most data is captured via that scan and fed directly into the computer system at the Processing Center, but all scans also are reviewed and quality-controlled by workers there.  The workers manually input any data not successfully captured through automated means.  The computer system at the Processing Center updates ASSETS during the next routine data exchange/batch processing session.

    8. ASSETS receives data from Financial Institution Data Matches (FIDM) monthly.
