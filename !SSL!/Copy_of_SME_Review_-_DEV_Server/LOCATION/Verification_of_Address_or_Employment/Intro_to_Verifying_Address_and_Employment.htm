Introduction to Verifying Respondent Address and Employment

The verification process (i.e. PCL and WHBR) can be manually or automatically initiated on the Case Mode > Status Information screen.

Verifying Location Information

Once obtained, PF/NCP addresses must be verified via a PCL form and Respondent employers must be verified via the Wage and Health Benefits Report form.

A PF/NCP's address may be found through an automated search or through manual, non-automated (both online and offline) location efforts.

What signifies the end of the location process?

Once the PF/NCP's Address and the PF/NCP's employer information are verified, location efforts have been completed, and ASSETS closes the L Group.

    • Verified PF/NCP addresses will appear on the Summary Mode > Resp Addr tab, and will contain Ver Ind code of Code M, Code R, or Code V .

    • Verified PF/NCP employers will appear on the Case Mode > Resp Emp tab, and will contain a Code V if the employer has been verified.

Note that automated searches of the National Directory of New Hires (NDNH)  and the State Directory of New Hires (SDNH) will continue in the background even with no L Group open.

This means that the verification codes for address and employer will appear in their proper places, and that there is no longer an L Group open on the case.

However, location efforts may be necessary multiple times over the life-cycle of a case.

 

A Code V in the Employer Verification field causes ASSETS to automatically generate an IWO, and signifies the end of Location efforts.  

In instances of New Hire Reporting, the Processing Center or ASSETS will enter the Code V automatically.

In all other instances, the worker must enter the Code V manually.
